# Story 1.2: Filament Admin Panel & Authentication

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As an **admin user (John)**,
I want **to log into a working Filament admin panel at `/admin` with sidebar navigation**,
So that **I can manage site content through a secure, authenticated interface**.

## Acceptance Criteria

**Given** Filament 5 is installed (`composer require filament/filament:"^5.0" -W`)
**When** I run `php artisan filament:install --panels`
**Then** the admin panel is accessible at `/admin`
**And** the admin panel uses the dark sidebar theme (#1e1b2e sidebar-bg) with correct admin color tokens (Inter typeface, #FF0000 primary)

**Given** I visit `/admin/login` while logged out
**When** I submit valid email + password credentials
**Then** I am redirected to the admin dashboard

**Given** I submit invalid credentials
**When** the login form validates
**Then** I see an error message and remain on the login page

**Given** I am not authenticated
**When** I visit any `/admin/*` URL
**Then** I am redirected to `/admin/login`

**Given** Filament is installed
**When** I check the sidebar navigation structure
**Then** it has group headings: Main (Dashboard, Services, Team, Blog, Pricing), Leads (Messages — v1.1 placeholder), Settings (Theme, Media Library, Pages)
**And** sidebar items use Blade Heroicons

## Dev Notes

### Critical Filament 5.7 Requirements

- **Filament 5.7 requires Tailwind CSS v4 and Livewire v4** — these are installed automatically as dependencies. DO NOT install Tailwind v3 or Livewire v3.
- **Install command:** `composer require filament/filament:"^5.0" -W` (the `-W` flag is required to upgrade/downgrade dependencies as needed).
- **Panel provider:** The `php artisan filament:install --panels` command auto-generates `app/Providers/Filament/AdminPanelProvider.php`. This is where panel configuration happens.
- **Filament Resources auto-discover** from each domain's `Filament/Resources/` directory — no manual registration needed.

### DDD Setup for Identity Domain

Create the Identity domain structure now (first DDD domain):
```
apps/backend/app/Domains/Identity/
├── Models/
│   └── User.php              # Can extend or alias Filament's built-in User model
└── Filament/
    └── Resources/
        └── UserResource.php  # Basic admin user management
```

**Important:** The default Laravel `User` model stays in `app/Models/User.php` but for DDD compliance, create a reference or extend it in the Identity domain. Filament's built-in auth uses the default `User` model.

### Admin Visual Identity (MANDATORY — match DESIGN.md exactly)

- **Sidebar BG:** `#1e1b2e`
- **Sidebar text:** `#a5a3b5`
- **Sidebar active:** `#FFFFFF` (with semi-transparent white bg)
- **Sidebar hover:** `rgba(255,255,255,.06)`
- **Primary color:** `#FF0000`
- **Surface BG:** `#f5f5f9`
- **Card BG:** `#FFFFFF`
- **Border:** `#e8e7ef`
- **Foreground:** `#222222`
- **Typeface:** Inter (Google Font) — all admin roles
- **Icons:** Blade Heroicons ONLY (NOT Font Awesome — that's for public site)
- **Sidebar width:** 260px

### Style Customization Approach

In Filament 5.7, customize through:
1. `AdminPanelProvider.php` — `colors()`, `font()`, `viteTheme()` or `theme()`
2. Custom CSS via Filament's theme system (`resources/css/filament/admin/theme.css`)
3. The `renderHook()` method for structural customizations

Example panel config:
```php
$panel
    ->default()
    ->path('admin')
    ->login()
    ->colors([
        'primary' => Color::hex('#FF0000'),
    ])
    ->font('Inter')
    ->navigationGroups([
        NavigationGroup::make('Main')
            ->icon('heroicon-o-home'),
        NavigationGroup::make('Leads')
            ->icon('heroicon-o-mail')
            ->collapsed(),
        NavigationGroup::make('Settings')
            ->icon('heroicon-o-cog-6-tooth')
            ->collapsed(),
    ])
```
